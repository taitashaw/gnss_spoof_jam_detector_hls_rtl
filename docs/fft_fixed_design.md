# From-scratch fixed-point FFT (`fft_fixed.hpp`)

Author: John Bagshaw — License: MIT (c) 2026 John Bagshaw

A self-contained synthesizable fixed-point FFT written to replace the Xilinx
`hls_fft.h` path, whose bit-accurate C-model aborts csim with an integer-divide
`SIGFPE`. Because this FFT is our own ordinary C++ (no vendor model), **csim and
cosim actually run** — the detector is now verified end to end against the Python
golden, which the FPE-blocked vendor path never allowed.

## Design

| Parameter | Choice |
|---|---|
| Algorithm | Radix-2 decimation-in-time (DIT), bit-reversed input load, natural-order output |
| Length | N = 2048 (11 stages); matches the ddMap config |
| Data type | `ap_fixed<24, 4, AP_RND_CONV, AP_SAT>` (4 integer bits, 20 fractional, convergent rounding, saturation) |
| Twiddle type | `ap_fixed<18, 2, AP_RND_CONV, AP_SAT>` (16 fractional bits; \|W\| ≤ 1) |
| Product accumulator | `ap_fixed<34, 6, AP_RND_CONV, AP_SAT>` (keeps the complex multiply before rounding) |
| Twiddle ROM | `W[k] = exp(-j 2πk/N)`, k = 0..N/2-1, generated by `scripts/gen_fft_twiddle.py` into `fft_twiddle_{re,im}.inc`; the double literals are converted to fixed point **at compile time** (a ROM) — no runtime floating point |
| Scaling | Per-stage divide-by-2 (total /N) — the classic "scaled" FFT — so values stay in range; output ≈ DFT/N |
| Rounding | `AP_RND_CONV` (convergent / round-to-even) on every result to remove bias |
| Inverse | Conjugate twiddles; the /N normalisation comes from the same per-stage halving |

The `/N` scale and any uniform rescale are harmless for acquisition: the correlation
**peak location** and the **early/late distortion ratio** are scale-invariant. (The
detector applies one `2^14` rescale to the spectral product so the triple-transform
correlation keeps full fixed-point precision — see `ddmap_sqm_hls.cpp`.)

## Accuracy vs numpy.fft (hard gate, `tb_fft_fixed.cpp`)

Forward FFT compared bin-by-bin against `numpy.fft.fft(x)/N`. **Hard PASS bound:
SNR ≥ 50 dB on every case.** Measured (plain csim / g++, no vendor model):

| Case | max bin error | RMS error | SNR | ENOB | Result |
|---|---|---|---|---|---|
| impulse | 0.00e0 | 0.00e0 | 527.8 dB | 87.4 | PASS (exact) |
| impulse7 | 8.45e-7 | 4.52e-7 | 54.66 dB | 8.8 | PASS |
| tone | 5.80e-6 | 1.65e-7 | 96.54 dB | 15.7 | PASS |
| random | 1.73e-6 | 5.63e-7 | 83.89 dB | 13.6 | PASS |
| C/A block | 1.74e-6 | 5.77e-7 | 85.64 dB | 13.9 | PASS |
| ifft(fft(x))·N round-trip | — | — | 68.05 dB | — | PASS |

The worst case (impulse7, 8.8 ENOB) is the flat-spectrum case where every golden bin
is uniformly tiny, so relative error is highest; the signal-bearing cases (tone,
random, C/A) sit at 13.6–15.7 ENOB — comfortable headroom over the ~5–6 bit input
quantisation, which is what GNSS code correlation needs. **The gate passed before the
FFT was wired into the detector.**

## Reproduce

```
python3 scripts/gen_fft_twiddle.py      # twiddle ROM (compile-time constants)
python3 scripts/gen_fft_vectors.py      # test vectors + numpy golden
g++ -std=c++14 -I hls/include -I $VITIS/include hls/tb/tb_fft_fixed.cpp -o fft_test
./fft_test .                            # the accuracy gate (no vendor model)
```
